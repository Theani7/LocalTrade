## Commands

### Backend
```bash
cd backend
npm install               # Install deps
cp .env.example .env      # Create env (fill in secrets)
npm run dev                # Start dev server (nodemon, port 5000)
npm test                   # Run tests (in-memory MongoDB, no external DB needed)
npm run test:watch         # Watch mode
npm run clear:data         # Reset DB (deletes all vendors/products/orders)
node seed-data.js          # Seed mock vendors + products
```

### Frontend
```bash
cd frontend/localtrade_app
flutter pub get            # Install Dart deps
flutter run                # Run on connected device/emulator
flutter build apk          # Build Android APK
flutter build ios          # Build iOS (macOS + Xcode required)
flutter analyze            # Lint check
```

## Key Gotchas

- **API base URL**: Hardcoded in `frontend/localtrade_app/lib/core/constants/app_constants.dart`. Default is the Render production URL. For local dev, switch to `http://10.0.2.2:5000/api/v1` (Android emulator) or `http://localhost:5000/api/v1`.
- **Tests require no external DB**: `npm test` uses `mongodb-memory-server` (in-memory). No MongoDB instance needed.
- **Backend uses Express 5**: Not Express 4. Route error handling differs (async errors propagate automatically).
- **Custom NoSQL sanitizer**: `app.js` has a custom sanitizer instead of `express-mongo-sanitize` due to Express 5 compatibility.
- **CommonJS modules**: Backend uses `"type": "commonjs"` ‚Äî use `require()`, not `import`.
- **Firebase config**: Frontend requires `firebase_options.dart` (generated by `flutterfire configure`). If missing, Firebase init is skipped gracefully.
- **Seed admin**: Default admin is `admin@gmail.com` / `admin123`. Run `node seed-data.js` to populate test data.

## Architecture Notes

- **Backend entry**: `src/server.js` ‚Üí `src/app.js`
- **API prefix**: All routes under `/api/v1/`
- **Auth**: JWT-based. Token stored in Flutter via `flutter_secure_storage`.
- **Roles**: `admin`, `vendor`, `customer`. Vendor approval workflow: pending ‚Üí approved/rejected.
- **Image uploads**: Cloudinary (not local storage).
- **Deployment**: Backend on Render (`render.yaml` at root). Frontend builds for Android/iOS.

## Testing

- Backend tests: `cd backend && npm test`
- Uses `jest` + `supertest` + `mongodb-memory-server`
- Setup file: `tests/setup.js` ‚Äî handles in-memory Mongo lifecycle
- Test timeout: 30s (configured in `jest.config.js`)
- Tests run sequentially (`--runInBand`)

### Key Design Rules
- **Color palette**: Cream background `#FBF5EA`, Coral `#FF6F52` (fills only, ink text ‚Äî never white on coral), Ink `#2B2620`, Muted `#6E6557`
- **Status badges**: Light-fill + dark-text pattern, never saturated fill with white text
- **Danger red `#D32F2F`**: Only for permanent delete. Reversible negative actions use outline buttons
- **Typography**: Inter or Noto Sans, 400/500 weights only, sentence case, min 12px for labels
- **Cards**: 16px radius, soft shadows `0 2px 10px rgba(43,38,32,0.05)`, padding 12-18px
- **Touch targets**: 44px minimum, 52px for primary actions
- **Tone**: Customer=warm/community, Vendor=supportive/practical, Admin=neutral/operational
